# CV-fix

A CV builder for jobseekers in Sweden's *Rusta och Matcha* employment program.
ATS-safe PDFs, mobile-first, with a coach mode for reviewing and editing participant CVs and optional AI assistance for profile text, experience bullets, skill suggestions, and job-posting keyword matching.

**Status:** MVP shipped 2026-03-15. AI assistance (PRD §15) shipped 2026-03-15 → 2026-03-20. See [PRD.md](./PRD.md) §5 for scope and §14 for V1.1 backlog.

---

## Stack

Next.js 14 (App Router) · TypeScript (strict) · Tailwind + shadcn/ui · Supabase (auth, RLS, Postgres) · React-PDF · React Hook Form + Zod · Anthropic SDK (`claude-sonnet-4-6`) · Vercel.

Tech stack is **locked** — see [PRD.md §4](./PRD.md) and [CLAUDE.md](./CLAUDE.md).

## Getting started

### Prerequisites

- Node 20+
- Supabase CLI: `brew install supabase/tap/supabase`
- GitHub CLI (for PRs): `brew install gh`
- Vercel CLI (for deploy): `npm i -g vercel`

### Local setup

```bash
git clone https://github.com/L-Lisa/CV-fix.git
cd CV-fix
npm install
```

Create `.env.local` at the repo root:

```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
NEXT_PUBLIC_SITE_URL=http://localhost:3000
ANTHROPIC_API_KEY=...
```

Link Supabase and pull types:

```bash
supabase login
supabase link --project-ref YOUR_PROJECT_REF
npx supabase gen types typescript --project-id YOUR_PROJECT_ID > types/database.ts
```

Run:

```bash
npm run dev          # http://localhost:3000
npx tsc --noEmit     # type check
npm run lint
```

## Repository layout

```
app/                 Next.js App Router (auth, app, guest groups + /api/*)
components/          UI components (cv-form, cv-preview, coach, pdf, ui, ...)
lib/
  actions/           Server Actions (auth, cv, coach, guest)
  queries/           Read helpers
  ats/               ATS validation (PRD §9)
  pdf/               React-PDF templates
  supabase/          Browser/server clients + middleware
  validation/        Zod schemas
supabase/migrations/ SQL migrations (one file currently)
types/index.ts       All app types — including AI* (PRD §15)
types/database.ts    Generated by supabase CLI — do not edit manually
```

API routes:

| Route | Purpose |
|---|---|
| `POST /api/cv/[id]/pdf` | Authed PDF export |
| `POST /api/cv/guest/pdf` | Guest PDF export |
| `POST /api/ai/profile` | AI: 3-sentence ATS profile (PRD §15.1) |
| `POST /api/ai/description` | AI: 3 strong-verb bullets for an experience |
| `POST /api/ai/skills` | AI: 6 skill suggestions |
| `POST /api/ai/keywords` | AI: missing keywords vs. job posting |
| `GET /auth/callback` | Email-confirmation redirect handler |

Coach-participant linking is a **Server Action** (`lib/actions/coach.ts`), not an API route.

## Deployment

Deployed on Vercel. The repo is linked (`.vercel/project.json` exists). Pushes to `main` trigger a deploy.

```bash
vercel login         # refresh CLI token
vercel               # preview deploy
vercel --prod        # production deploy (only if you know what you're doing)
```

Set the same env vars in Vercel → Project → Settings → Environment Variables.
